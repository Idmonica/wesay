Welcome to WeSay 0.7.x-Development

You have installed a development version of WeSay.  This is a team effort, and we really appreciate your willingness to give WeSay a spin and send us feedback. 

About this version

WeSay comes in two flavors: “stable” (even numbered ones) and “development” (odd numbered ones).   The even versions receive critical bug fixes, but not new features or anything risky.

If crashes or other bugs would be a major problem for you, please uninstall this version and get our most recent 0.6.xxxx one.  If you are helping us advance WeSay, are evaluating WeSay’s capabilities, or are just willing to risk the occasional bug, this is the version for you.  This is where we are currently adding new abilities in response to your requests, and we need people to help ensure that it can become stable, too. Eventually, we will retire 0.6 and 0.7 will be relabeled “0.8 - Stable”. Note that at times we post new “odd” versions almost daily.  If you report a bug via issues@wesay.org, we can often times fix it and provide a new release within hours.  So let us know if you’re in a hurry for a fix.

What’s New in 0.7?

Here’s what we’ve added, so far, to 0.7.  More improvements are planned.

Team Collaboration using Send/Receive & Chorus

The plan for the 0.7 series is to enable multiple team members to work on the same dictionary.  This is done using something called Chorus along with Mercurial… it will come in stages, beginning with this version, which provides teams with the bare minimum needed to synchronize their work using USB keys, office networks, or a server on the internet. To use these features, you must download and install either Mercurial or TortoiseHg onto each machine which will participate.

What Now?

Get your head around WeSay… it’s really different.  It’s aimed at empowering a very different user than you are, and approaches the task of dictionary making in a new, super-task-oriented way.  Using the provided samples, play around with both WeSay and the WeSay Configuration Tool.  To do this, go to your Windows Start menu and find “WeSay”.  Choose one of the icons there which open the sample project.

Documentation

Currently, we keep the documentation as a separate download.  You can get it and read more advanced help topics on our wiki, here.

Ready to go deeper?

Consider joining the Google Group. You can either use this group from your browser, through your email client, or a mix of both.  Until we have real documentation, this is the place to go when something is unclear.  As you ask questions, we’ll answer them there.  A Google Group is a mailing list, but one that you can choose to have sent to your email box or wait for you to browse on the web, your choice. You can read old questions and answers.

Please report a bugs or request a feature using email. Tell us that you tried it and what you did.  No, really.  It’s human nature to assume someone else had the same crash… but if it was easy to run into, we would have found it, and we would have fixed it.    When you send to this address, your issue is automatically entered into our "todo list" and you will get updates as we work on the problem.

We are adding to WeSay's abilities weekly, and publishing new versions.  Major new features are always announced on the blog.
