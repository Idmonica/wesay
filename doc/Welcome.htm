Welcome to WeSay 0.6.x-Stable

You have installed a stable version of WeSay.  This is a team effort, and we really appreciate your willingness to give WeSay a spin and send us feedback. 

About this version

WeSay comes in two flavors: “stable” (even numbered ones) and “development” (odd numbered ones).   The even versions receive critical bug fixes, but not new features or anything risky.

If crashes or other bugs would be a major problem for you, then this is the release for you.  However, if you would like to evaluate WeSay’s capabilities or are willing to risk the occasional bug, then please download one of our 0.7.x – development versions which is where we are currently adding new abilities in response to your requests, and we need people to help ensure that it can become stable, too. Eventually, we will retire 0.6 and 0.7 will be relabeled “0.8 - Stable”. Note that at times we post new “odd” versions almost daily.  If you report a bug via issues@wesay.org, we can often times fix it and provide a new release within hours.  So let us know if you’re in a hurry for a fix.

What’s New in 0.6?

Here’s what we’ve added.

Enter Example Translations in Add Missing Example Sentences Task

In the “Add Missing Example Sentences” task, You can now enable a place for immediately collecting a translation of the example.  (This was always possible in the Browse & Edit view, of course).

Add translation in Gather By Semantic Domain Task

You can now turn on an option to show a “meaning” field in the Gather By Semantic Domains task.   Be warned, this has some problematic aspects.  For example, if you later click on a word you previously entered, and change its spelling, you will now get a new word, and the old spelling will remain.  Some of this may seem like a “bug” to a human, but it’s hard for the program to safely guess what the right thing is to do, and the “WeSay way” is to avoid asking the user questions.  We welcome your ideas on this. In the meantime, we leave it to you to decide if the benefits outweigh the problems.

Semantic Domain Numbers

By popular demand, numbers have returned, everywhere we show Semantic Domain labels.  This is a good time to note that if you want to add semantic domains in the Browse & Edit view, you can do that by typing in the number of the domain, or search for an applicable domain by typing in relevant words.

Linux

We’ve got WeSay running on Ubuntu Linux, so if you would like to help us test that version too, let us know.  We even have SD chips which boot an OLPC XO into Ubuntu so you can run WeSay in indestructible 5 watt bliss.  Some features are as yet unavailable on Linux, including audio features, and pdf dictionary making. Check out the WeSay Linux Known Issues wiki for more information.

Peace of Mind

An important new feature we’ve adding in 0.6 is automated incremental backup.  The way this works is that you put some kind of memory stick into the laptop (like the memory your digital camera uses), then tell WeSay where to find it.  You just leave that memory chip in, permanently.  WeSay will silently keep changes there, in a way that you can always “roll-back” if things go very wrong.  This is the first feature that uses the Chorus collaboration component SIL is building.  To enable this and forthcoming Chorus features, you need to install a package called “Mercurial”. On Windows, download and install TortoiseHG.

One click to Create PDF Dictionary

A PDF dictionary is now a mouse-click away, once you install princexml.  (This feature is Windows only, temporarily). Please, >let us know if you use this feature.

Open With Lexique Pro

If you need more some more than the basics, WeSay now provides a button for opening your dictionary in Lexique Pro (version 3) (Windows only).  See the blog for details.

Audio Recording and Playback

You can now add audio recording and playback to any text field.  Simply create a new writing system, and mark it “Audio”.  Then tick this writing system for any fields (e.g. Word or Example) where you want to enable users to speak the text, as well as write it.  Audio files are created in the new “audio” folder of the WeSay project.  (Temporarily Windows only). See the blog for details.  Please, let us know if you use this feature.

Where was I?

WeSay now remembers where you were in the dictionary browse & edit.  It also remembers any adjustments you made to change the size of parts of the screen.

Art of Reading Integration

Now WeSay will use definitions to search SIL's Art Of Reading clipart library, if you have that CD in the drive or have copied the contents to C:\ (on windows).  A box opens up with images that might fit the entry.  If you choose one, it is copied to the dictionary project and inserted in the entry.  See the blog for details.

Faster

We're always looking for ways to make WeSay fast even on small, inexpensive computers.  Recent 0.6 builds are our fastest ever.

Save for FieldWorks 5.4

The LIFT xml format which WeSay uses continues to improve. This version of WeSay now conforms to version 0.13 of the standard, as does the forthcoming FLEx 6.0. However, since FLEx 6.0 won't be publically avialable for several months, this raises the problem for people wishing to load WeSay data into FieldWorks Language Explorer 5.4. That version of FLEx will tell you that it doesn't know how to read LIFT version 0.13. We've added a command, found on the toolbar in the WeSay Configuration Tool, to "Save a copy for FLEx 5.4".

Easier import from FLEx

By popular arm-twisting ;-), it is now easier to start a WeSay project from a FLEx LIFT export. When you run WeSay Configuration Tool, there is now an option to "Create Project From FLEx LIFT Export". WeSay will look through the writing systems in the LIFT, adding them to the WeSay Configuration, and set the Writing Systems to show for a few common fields, based on what it finds in the LIFT file. You will still need to go in an set the fonts, sort order, and keyboard for Writing Systems. When FieldWorks 6 arrives, we should be able to import that information as well.

What Now?

Get your head around WeSay… it’s really different.  It’s aimed at empowering a very different user than you are, and approaches the task of dictionary making in a new, super-task-oriented way.  Using the provided samples, play around with both WeSay and the WeSay Configuration Tool.  To do this, go to your Windows Start menu and find “WeSay”.  Choose one of the icons there which open the sample project.
