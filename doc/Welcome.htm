Welcome to WeSay 0.9.x-Development

You have installed a development version of WeSay.  This is a team effort, and we really appreciate your willingness to give WeSay a spin and send us feedback. 

About this version

WeSay comes in two flavors: 'stable' (even numbered ones) and 'development' (odd numbered ones).   The even versions receive critical bug fixes, but not new features or anything risky.

If crashes or other bugs would be a major problem for you, please uninstall this version and get our most recent 0.8.xxxx one.  If you are helping us advance WeSay, are evaluating WeSay's capabilities, or are just willing to risk the occasional bug, this is the version for you.  This is where we are currently adding new abilities in response to your requests, and we need people to help ensure that it can become stable, too. Eventually, we will retire 0.8 and 0.9 will be relabeled '1.0 - Stable'. Note that at times we post new 'odd' versions almost daily.  If you report a bug via issues@wesay.org, we can often times fix it and provide a new release within hours.  So let us know if you're in a hurry for a fix.

What's New in 0.9?

Printing

WeSay has long provided one-click creation of PDF dictionaries. If you don't like the default layout, you can even use FLEx to make custom CSS stylesheets that WeSay will use, so that it 'just works' for the WeSay user.  However, that method does not work for some languages with complex non-roman scripts. So now WeSay provides the option to create an OpenOffice Writer document.  This removes the previous limitations, since Open Office includes SIL's advanced Graphite font technology.

Increased Standardization

The Palaso team is continually working to make WeSay conform to various file standards in order to ease integration with other tools you use, like FLEx (FieldWorks Language Explorer).   With this release, WeSay now uses industry-standard LDML ('Locale Data Markup Language') to store writing system information, rather than its own xml format.  You'll only notice a difference if you're exchanging data with FLEx, which has also switched to this format.

What Now?

Get your head around WeSay' it's really different.  It's aimed at empowering a very different user than you are, and approaches the task of dictionary making in a new, super-task-oriented way.  Using the provided samples, play around with both WeSay and the WeSay Configuration Tool.  To do this, go to your Windows Start menu and find 'WeSay'.  Choose one of the icons there which open the sample project.
